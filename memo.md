# FPSベースゲーム

## 概要

- html/js/cssで動くFPSを作成してください。
- ゲーム性やルールなどは作っていきながら考えたいので、とりあえずFPSのベースとなる物、シンプルで最低限のFPSを作りたいです。
- グラフィックはQuake3ぐらいで良いです。
- 武器はシンプルにマグナム1つだけで、頭に当てたら一撃即死。胴体は2発、それ以外は5発で死にます。
- マップはシンプルなアリーナ型で。
- WASD操作でスペースでジャンプです。
- 敵はひとまずCPUで。

### 実装を進めるにあたって

- この項目の内容は全て永続的で絶対の指示となる。
- 基本的な流れとしては、「memo.mdを確認 -> 実装 -> 動作確認 -> memo.mdに改善点、修正点を記述」のループになります。
- memo.mdに記入出来るのは私だけです。あなたはmemo.mdを読むことしか出来ません。
- 私が「いつもの」と言ったら、memo.mdの最新の指示を確認し、それに従いましょう。
- 将来的なことも考え、綺麗でわかりやすいコードを心がけましょう。
- 処理にはわかりやすい日本語コメントを付けましょう。
- デバッグしやすいように逐一日本語デバッグログを入れるようにしましょう。
- 後々機能追加などがしやすいように、処理ごとにモジュール化するようにしましょう。
- 動的言語を使用する時も型ヒントやjsdocなど型を守らせる仕組みは必須となります。型テストも必ず行いましょう。
- 実装/改修時は./docsフォルダの中に.md形式の詳細設計書を作成/修正するようにしましょう。
- 詳細設計書の名前はそのファイルの名前を使えってください。例えば、「aaa.sh」というスクリプトの詳細設計書の名前は「aaa.sh.md」になります。
- 日付を入力する際は必ずコマンドで取得してからにしてください。
- 実装したらテストも必ず行うようにしてください。テストが通るまで完成にはなりません。
- 実装/改修を進める前に必ずその内容を説明して、許可を取ってから進めるようにしてください。
- 一通り実装、動作確認が完了したら、最後に操作手順書を「README.md」という形式で作成してください。
